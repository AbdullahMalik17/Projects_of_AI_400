# Deployment Guide

This guide outlines how to deploy the **AI-Powered Task Management System** for production or demonstration purposes.

## Architecture Overview

- **Frontend**: Next.js (Deploy to Vercel)
- **Backend**: FastAPI (Deploy to Render/Railway/Heroku)
- **Database**: PostgreSQL (Deploy to Neon/Render/Supabase)
- **AI**: Google Gemini API (External Service)

---

## 1. Database Setup (Neon PostgreSQL)

We will use **Neon** for a free serverless PostgreSQL database.

1.  Go to [Neon.tech](https://neon.tech/) and sign up.
2.  Create a new project (e.g., `ai-task-manager`).
3.  Copy the **Connection String** (e.g., `postgres://user:pass@ep-xyz.aws.neon.tech/dbname?sslmode=require`).
4.  **Important**: Save this string; you will need it for the Backend environment variables.

---

## 2. Backend Deployment (Render)

We will use **Render** to host the FastAPI backend.

1.  **Push your code to GitHub**. Ensure your repo contains the `AI_Powered_Task_Management_System` folder.
2.  Sign up at [Render.com](https://render.com/).
3.  Click **New +** -> **Web Service**.
4.  Connect your GitHub repository.
5.  **Configure the Service**:
    *   **Name**: `ai-task-manager-backend`
    *   **Root Directory**: `AI_Powered_Task_Management_System/backend`
    *   **Environment**: `Python 3`
    *   **Build Command**: `pip install -r requirements.txt`
    *   **Start Command**: `uvicorn app.main:app --host 0.0.0.0 --port 10000`
6.  **Environment Variables** (Scroll down to "Environment Variables"):
    *   `PYTHON_VERSION`: `3.10.0` (or your local version)
    *   `DATABASE_URL`: *(Paste your Neon Connection String here)*
    *   `GEMINI_API_KEY`: *(Your Google Gemini API Key)*
    *   `JWT_SECRET_KEY`: *(A long random string, e.g. generated by `openssl rand -hex 32`)*
    *   `ENVIRONMENT`: `production`
    *   `ALLOWED_ORIGINS`: `https://your-frontend-app.vercel.app,http://localhost:3000` (You will update this after deploying frontend).
7.  Click **Create Web Service**.
8.  Wait for the build to finish. Copy your **Backend URL** (e.g., `https://ai-task-manager-backend.onrender.com`).

---

## 3. Frontend Deployment (Vercel)

We will use **Vercel** to host the Next.js frontend.

1.  Go to [Vercel.com](https://vercel.com/) and sign up.
2.  Click **Add New...** -> **Project**.
3.  Import your GitHub repository.
4.  **Configure the Project**:
    *   **Framework Preset**: Next.js
    *   **Root Directory**: Edit this and select `AI_Powered_Task_Management_System/frontend`.
5.  **Environment Variables**:
    *   `NEXT_PUBLIC_API_URL`: `https://ai-task-manager-backend.onrender.com/api/v1` (The URL from Step 2 + `/api/v1`).
6.  Click **Deploy**.

---

## 4. Final Configuration

1.  Once the Frontend is deployed, copy its URL (e.g., `https://ai-task-manager-frontend.vercel.app`).
2.  Go back to **Render** (Backend Dashboard).
3.  Update the `ALLOWED_ORIGINS` environment variable to include your new Frontend URL.
    *   Example: `https://ai-task-manager-frontend.vercel.app,http://localhost:3000`
4.  **Redeploy** the backend (Manual Deploy -> Deploy latest commit) to apply the CORS change.

---

## 5. Verification

1.  Open your Vercel URL.
2.  Go to the **AI Assistant** tab.
3.  Type: *"Create a test task to verify deployment."*
4.  If the task is created and the agent responds, your full stack application is live!
